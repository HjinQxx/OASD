# Development Principles

이 프로젝트는 사용자 맞춤 지도 앱 `personal_map`을 안정적으로 확장하기 위해 아래 원칙을 따릅니다.

## 1. 기술 스택

- Vite를 기반으로 개발합니다.
- React를 사용해 UI를 구현합니다.
- TypeScript를 사용해 타입 안정성을 유지합니다.

## 2. 외부 서비스 제한

- 외부 생성 서비스나 별도 서버를 사용하지 않습니다.
- 기능 구현은 가능한 한 프로젝트 내부 코드와 클라이언트 로직을 중심으로 진행합니다.

## 3. 단계별 구현

- 한 번에 큰 기능을 만들지 않습니다.
- 작은 단위로 기능을 나누고, 각 단계가 동작하는지 확인하면서 확장합니다.
- 새로운 기능은 기존 구조와 스타일을 먼저 확인한 뒤 추가합니다.

## 4. 구현 후 확인

- 기능 구현 또는 수정 후에는 `npm run build`로 빌드가 통과하는지 확인합니다.
- 빌드 오류가 발생하면 원인을 확인하고 수정한 뒤 다시 검증합니다.

## 5. 삭제와 큰 구조 변경

- 파일 삭제, 주요 폴더 이동, 큰 구조 변경을 하기 전에는 먼저 변경 의도와 영향을 설명합니다.
- 사용자의 확인이 필요한 변경은 임의로 진행하지 않습니다.
